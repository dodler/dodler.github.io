This is a simple comprehensive guide how to setup Teiid with simple and single datasource. We will use Postgresql as db. I am using Ubuntu 16.04 but it doesn’t make big difference.

So, what we want to do? We need to get data from our target db from designer using teiid.

At first let’s setup postgres. You can install it simply using apt-get command. At first you should create database. Let’s say it would be Test database:

create database Test;

Then establish connection:

\connect Test;

So, you are connected to our new db, and now you need to create a test table in it. Let’s say it would be purchases with field id, name, amount, price.

create table purchases(

id integer primary key,

name varchar(20),

amount integer,

price amount float8);

For now we have db ready.

First to do is download Teiid designer. This is a plugin for Eclipse, so, you need to download both of them. Very easy and comprehensive manual at this point you can find here. After you got  it installed and Eclipse restarted don’t forget to show Teiid Designer perspective.

For Teiid you will need a server. You can install it separately, or download using link proposed by Teiid Designer. I recommend to use Teiid Designer for server installation, because otherwise you will need to setup location. This is not some sort of rocket science, but this will take some time. And installation via designer is much easier.

So, you have designer with server installed. I have workspace like this one.

On the right side you can see guide how to define and edit vdb. You can follow it to get data from designer.

So, create new model project. Define Teiid Model Project -> New, input some name (I used teiid_purchases), Finish -> Ok. Now you will see project name on the opposite to Define Teiid Project Model menu.

Then create JDBC connection to postgres. Double click to Create JDBC Connection, choose PostgreSQL in drop         down menu, press next and your database name (test).

When you press “Test Connection” your connection should be successfull. Then click finish, and voila, you now have working JDBC connection.

Now it’s time to create new source model for newly established JDBC connection. Click “Create source model”. In menu that appears choose recently created connection and click next. Click next one more time and you will such window.

In public tables you need to find table, that you recently have created. Choose it and click next and then finish.

Before you start you need to start server. Open ‘Servers’ tab and you will see this.

Server is stopped. Select it and Ctrl+Alt+R it. You will see many startup logs and finally server status will switch to ‘Started’.

Now right click on purchases -> Modelling -> Preview Data and now check out SQL Results tab. It will contain records from purchases table. But we didn’t add any records, so for now this will be empty. Try to add something and you will see changes.

Next step is to create and deploy Virtual DataBase on server. Some basic understanding of vdb you can find here.

Press ‘Define VDB’ button at guides section. You will see a VDB creation menu like this one:

I’ve already created vdb, but your will be empty. Then hit ‘New’ button and you will see the following window:

Choose project folder, input some vdb name (I did ‘test’), and add then model, you’ve created on previous step. Then hit finish and you’ve done with vdb creation. One more thing remaining. After vdb is created, you will see this window.

But you will have version like vdb name(I had test). Change it to some integer and you will be able to deploy vdb on server without any problems.

Next step is ‘Edit VDB’, but this one is optional, you can directly do ‘Execute VDB’. Once you’ve done this step, sql editor window will be opened and you will be able to execute scripts to vdb.
