One more interesting question is compare performance of raw PostgreSQL and VDB based on same postgres table.

I build 2 web services using Spring. They both do simple sql request - ‘select * from purchases;’.One uses postgres jdbc driver, second - teiid’s one. I will extend list of sql requests in future.

‘purchases’ table contains 4 fields - integer id, string name, integer price and integer amount.

Testing was performed using JMeter - this is Apache open-source performance measure tool. More info you can find here. JMeter was configured to execute with 100 number of threads and forever loop count.

I was testing system on my laptop with Intel Core i3 6006u processor.

I got almost equal throughput during measurement on over million requests. I got 4062 op/sec on postgres db and 4065 op/sec on vdb based on same table.
